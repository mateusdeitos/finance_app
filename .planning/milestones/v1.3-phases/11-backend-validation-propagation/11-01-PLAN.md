<objective>
Replace the blanket "child transaction cannot be updated" validation with field-level checking that allows date, description, category, tags, and propagation on linked transactions while rejecting all other fields. Guard the propagation loop so date/description changes do NOT cross-propagate to the partner's linked transactions when editing a linked transaction.

Purpose: Implements the core backend logic for editing linked transactions — validation (VAL-01, VAL-02) and propagation scoping (PROP-01).
Output: Modified errors.go with new error constant, modified transaction_update.go with validation and propagation changes.
</objective>
