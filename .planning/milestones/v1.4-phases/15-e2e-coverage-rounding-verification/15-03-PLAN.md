```yaml

        working-directory: frontend
        run: npx playwright install --with-deps chromium
```

Non-negotiables:
  - **Position:** BEFORE Playwright install — unit tests don't need chromium or the docker stack (saves ~60 seconds when unit tests fail fast).
  - **Working directory:** `frontend` — `npm run test:unit` is defined in `frontend/package.json`.
  - **No new env block needed:** unit tests are pure-function; no BACKEND_URL / PLAYWRIGHT_BASE_URL required.
  - **No `if: always()`:** unit-test failure SHOULD block the Playwright run (fail-fast per D-CI-1).
  - **No `timeout-minutes:` needed:** node:test with 6 cases runs in <1s; the default 6h job timeout is fine.
  - **Do NOT reorder any other step.**
  - **Do NOT create a separate workflow file** (`test-frontend.yml`) — PATTERNS.md §Option A / CONTEXT.md §D-CI-1 both recommend single-workflow.
  - **Do NOT add `.github/workflows/**` to the `paths` filter** — unnecessary, and could retrigger on unrelated workflow edits.

Two-space indentation for YAML keys matches the surrounding file — match it exactly. Do NOT introduce tabs.
