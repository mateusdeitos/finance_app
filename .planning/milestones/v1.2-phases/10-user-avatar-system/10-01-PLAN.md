## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OAuth provider -> auth_handler | AvatarURL string from external provider crosses into backend |
| Client -> account API | avatar_background_color hex value from user input |
| Account Search SQL -> client | partner_avatar_url exposed to requesting user |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | avatar_background_color | mitigate | Frontend enforces via ColorSwatchPicker preset list; backend column is VARCHAR(7) with NOT NULL constraint; full hex validation added in Plan 03 AccountForm zod schema `z.string().regex(/^#[0-9a-fA-F]{6}$/)` |
| T-10-02 | Spoofing | avatar_url from OAuth | accept | URL is stored server-side from trusted OAuth provider response; rendered as img src only, not used for redirects; degrades to initials on failure |
| T-10-03 | Info Disclosure | partner_avatar_url | accept | Account Search already filters by `user_id IN options.UserIDs`; partner URL only visible to users in the same accepted connection; same trust boundary as existing connection data |
</threat_model>
