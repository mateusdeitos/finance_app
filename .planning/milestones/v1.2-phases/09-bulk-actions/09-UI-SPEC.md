## Spacing Scale

Declared values — Mantine's built-in spacing tokens map to multiples of 4:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, between radio options, bullet list padding |
| sm | 8px | Gap inside Group rows, compact stack items |
| md | 16px | Default Stack gap, drawer body section gap |
| lg | 24px | Drawer section separators |
| xl | 32px | Drawer body bottom padding |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: Touch targets for menu items and action buttons must be minimum 44px tall (mobile-first requirement from `frontend/CLAUDE.md`). Toolbar bar padding: 0.75rem top/bottom, 1rem left/right (matches existing `SelectionActionBar.module.css`).

## Typography

Mantine type scale applied consistently with existing components. Exactly 2 font weights are used in this phase.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (sm) | 400 (regular) | 1.5 |
| Label / count | 14px (sm) | 700 (bold) | 1.5 |
| Drawer title | 16px (md) | 700 (bold) | 1.2 |
| Dimmed / secondary | 14px (sm) | 400 (regular) | 1.5 |

Weights: **400** (body, dimmed) and **700** (bold labels, drawer titles). No intermediate weight (600) is used.

Source: Existing component patterns in `BulkDeleteProgressDrawer.tsx`, `PropagationSettingsDrawer.tsx` — replicate exactly.

The "Excluir" action within the menu uses `color="red"` only on the icon, not on the menu item text.

## Component Inventory

New and modified components for this phase:

### Modified: SelectionActionBar

**Current state:** Single "Excluir" button (`Button color="red"`).

**New state:** Mantine `Menu` component replacing the button.

Layout: `[X close icon] [count text] [Ações menu button]`

Primary visual anchor: the "Ações" menu button is the focal point of the bar. The adjacent count text (e.g. "3 selecionadas") provides noun context that makes the bare verb label "Ações" unambiguous.

Menu items (in order):
1. "Alterar categoria" — `IconCategory` or `IconTag` icon (left section, 14px)
2. "Alterar data" — `IconCalendar` icon (left section, 14px)
3. `Menu.Divider`
4. "Excluir" — `IconTrash` icon (left section, 14px), `color="red"` on icon only

Menu button label: "Ações" with `IconChevronDown` or `IconDots` right section, size="sm", variant="default".

Props added to `SelectionActionBarProps`: `onCategoryChange: () => void`, `onDateChange: () => void`. The `onDelete` prop remains.

### New: SelectCategoryDrawer

A read-only variant of `CreateCategoryDrawer`. Opens from the right (`position="right"`, `size="md"`).

Purpose: Display category list for selection only — no create, no edit, no emoji edit.

Behavior: User taps a `CategoryCard` → drawer closes with selected `Transactions.Category`. Cancel/dismiss rejects the promise.

Return type: `Transactions.Category` (resolves via `renderDrawer<Transactions.Category>`).

Title: "Selecionar categoria"

Empty state: `<Text c="dimmed" size="sm">Nenhuma categoria cadastrada</Text>`

No action buttons at bottom — tapping a card is the confirm action.

### New: SelectDateDrawer

A bottom drawer (`position="bottom"`) for picking a date.

Styles (match existing bottom drawers):
```
content: { borderRadius: 'var(--mantine-radius-lg) var(--mantine-radius-lg) 0 0', height: 'auto', maxHeight: '80dvh' }
body: { paddingBottom: 'var(--mantine-spacing-xl)' }
```

Content:
- `DateInput` from `@mantine/dates` with `locale="pt-BR"`, `valueFormat="DD/MM/YYYY"`, default value = today
- "Aplicar" button below (`size="sm"`, Mantine default primary color, full width or `alignSelf: 'flex-start'`)

Return type: `Date` (resolves via `renderDrawer<Date>`).

Title: "Alterar data"

### New: BulkProgressDrawer (generalizes BulkDeleteProgressDrawer)

Generic version that accepts an action function instead of hardcoding `deleteTransaction`.

Props interface:
```ts
interface BulkProgressDrawerProps<T> {
  items: T[]
  getLabel: (item: T) => string
  action: (item: T) => Promise<void>
  title: {
    processing: string
    success: string
    error: string
  }
  successMessage: (count: number) => string
  onInvalidate: () => void
  onSuccess: () => void
}
```

States: `processing` | `success` | `error` — identical behavior pattern to `BulkDeleteProgressDrawer`.

Processing state:
- Animated `Progress` bar, `color="blue"`, `data-testid="bulk_progress_bar"`
- Dimmed centered label: current item description, `data-testid="bulk_current_label"`

Success state:
- `ThemeIcon color="teal"` + `IconCheck` (18px)
- Success message text (count-aware)
- "Fechar" button (`variant="default"`)

Error state:
- `ThemeIcon color="red"` + `IconX` (14px) + failed item description
- Dimmed error reason text
- Remaining items list (if any), each prefixed with `•`
- "Fechar" button (`variant="default"`)

Position: `position="bottom"`. Close disabled during processing state (same pattern as existing drawer).

### Modified: PropagationSettingsDrawer

Reused as-is for bulk category and date change actions. The labels inside the drawer currently read "Excluir" — these must be generalized to "atualizar" for non-delete actions.

**Option A (preferred):** Pass an `actionLabel` prop: `string` to the drawer. Default `'excluir'` for delete, `'alterar'` for category/date changes. Radio description strings become dynamic.

**Option B:** Create a new `BulkPropagationSettingsDrawer` with updated copy.

Executor should choose Option A (less duplication). Claude's discretion as noted in CONTEXT.md D-08.

### Fix: Bottom Drawer Content Centering (D-07)

All bottom drawers must center their content while the drawer itself remains full width.

Apply to: `SelectDateDrawer`, `BulkProgressDrawer`, `PropagationSettingsDrawer`.

CSS fix: on the Mantine Drawer `body` styles, do NOT apply `align-items: center` at the drawer level. Content Stack components should use `align="center"` only for success/error states (matching existing `BulkDeleteProgressDrawer` pattern). The fix in D-07 refers to ensuring the drawer body does not produce off-center text on wider screens — apply `maxWidth: '480px'` + `margin: '0 auto'` on the inner `Stack` when rendering on desktop viewports, or use Mantine `Container` with `size="xs"` inside the drawer body.

## Interaction Flow Contract

### Flow 1: Bulk Category Change

1. User has transactions selected (`selectedIds.size > 0`) → `SelectionActionBar` visible
2. User taps "Ações" → Mantine Menu opens
3. User taps "Alterar categoria" → `onCategoryChange()` called on page
4. Page calls `renderDrawer<Transactions.Category>(() => <SelectCategoryDrawer />)`
5. If user selects a category: promise resolves with category
6. Page checks `hasRecurring` (any selected tx has `transaction_recurrence_id`)
7. If yes: `renderDrawer<PropagationSetting>(() => <PropagationSettingsDrawer actionLabel="alterar" />)`
8. Page filters `selectedIds` — exclude txs where `original_user_id !== currentUserId` (silent skip, SEL-02)
9. Page builds item list: `{ id, description, categoryId, propagationSettings? }`
10. Page calls `renderDrawer(() => <BulkProgressDrawer items={...} action={updateTransaction} ... />)`
11. Drawer processes sequentially, stops on first error
12. On completion (success or error): `onInvalidate()` then `onSuccess()` (clear selection)

### Flow 2: Bulk Date Change

Same as Flow 1, replacing step 4 with `renderDrawer<Date>(() => <SelectDateDrawer />)`.

### Flow 3: Bulk Delete (unchanged)

Existing `handleDeleteClick` flow — `BulkDeleteProgressDrawer` continues to work independently until executor decides to migrate it to the generic `BulkProgressDrawer`. Migration is optional. Confirmation is handled by the existing `handleDeleteClick` flow (no new confirmation UI needed for this phase).
