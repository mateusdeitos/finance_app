## Directory Layout

```

│   ├── components/             # Shared/reusable UI components

│   │       └── import/         # CSV import review components

```

## Directory Purposes

**`src/api/`:**
- Purpose: One file per backend resource domain; plain async functions only
- Contains: `fetch` calls, URL construction, error throwing (`throw res`), type casts
- Key files:
  - `src/api/transactions.ts` — all transaction endpoints including CSV import and duplicate check
  - `src/api/auth.ts` — `fetchMe()`, `logout()`; defines `Me` type locally
  - `src/api/accounts.ts`, `src/api/categories.ts`, `src/api/tags.ts`, `src/api/userConnections.ts`

**`src/hooks/`:**
- Purpose: TanStack Query wrappers; one file per operation
- Contains: Query hooks returning `{ query, invalidate }`, mutation hooks returning `{ mutation }`
- Key files:
  - `src/hooks/useTransactions.ts` — primary transaction list query
  - `src/hooks/useCreateTransaction.ts` — handles field error mapping on failure
  - `src/hooks/useUpdateTransaction.ts` — plain mutation, no built-in invalidation
  - `src/hooks/useMe.ts` — supports `select` transformer; staleTime 5 minutes
  - `src/hooks/useTransactionPrefill.ts` — persists last-used date/category/account for new transactions
  - `src/hooks/useActiveFilters.ts` — reads filter state from route search params
  - `src/hooks/useIsMobile.ts` — breakpoint detection for mobile-first branching

**`src/components/transactions/form/`:**
- Purpose: All form-related components and schemas for transaction create/edit/import
- Key files:
  - `TransactionForm.tsx` — main form body; uses `useFormContext<TransactionFormValues>`
  - `RecurrenceFields.tsx` — recurrence switch/select/date/repetitions; accepts `namePrefix` for import reuse
  - `SplitSettingsFields.tsx` — expense split rows; uses `useFieldArray`; accepts `namePrefix`
  - `CurrencyInput.tsx` — controlled cents input
  - `DescriptionAutocomplete.tsx` — description field with suggestion typeahead
  - `transactionFormSchema.ts` — `transactionFormSchema`, `baseTransactionFields`, `applySharedRefinements`, `TransactionFormValues`
  - `importFormSchema.ts` — `importFormSchema`, `importRowFormSchema`; extends `baseTransactionFields`

**`src/components/transactions/`:**
- Key files:
  - `CreateTransactionDrawer.tsx` — initializes form with defaults + prefill; opened via `renderDrawer`
  - `UpdateTransactionDrawer.tsx` — initializes form from existing transaction; includes `UpdatePropagationSelector` for recurring
  - `TransactionList.tsx` — renders grouped transaction rows
  - `TransactionRow.tsx` — individual row; opens `UpdateTransactionDrawer` on click
  - `TransactionFilters.tsx` — filter bar; reads/writes route search params
  - `PropagationSettingsDrawer.tsx` — awaited drawer for bulk delete propagation selection
  - `BulkDeleteProgressDrawer.tsx` — progress display during bulk delete

**`src/routes/`:**
- Purpose: Page-level route definitions; file names map to URL paths
- Key files:
  - `__root.tsx` — root context (QueryClient injection)
  - `_authenticated.tsx` — auth guard + `<AppLayout>` shell
  - `_authenticated.transactions.tsx` — main transactions page; search param schema with Zod
  - `_authenticated.transactions_.import.tsx` — CSV import review page
  - `_authenticated.accounts.tsx`, `_authenticated.categories.tsx`
  - `login.tsx` — OAuth login page
  - `auth.callback.tsx` — OAuth callback handler
  - `_authenticated.connect-with.$externalId.tsx` — user connection invite acceptance

**`src/types/`:**
- Purpose: Domain type definitions in namespaces; one file per backend resource (currently only `transactions.ts` exists)
- Key file: `src/types/transactions.ts` — `Transactions` namespace with all transaction-related types and payload interfaces

**`src/utils/`:**
- Purpose: Pure functions and shared infrastructure; no React, no hooks
- Key files:
  - `queryKeys.ts` — `QueryKeys` const (must be used in all `queryKey` arrays)
  - `renderDrawer.tsx` — imperative drawer factory; `useDrawerContext<T>()` hook
  - `buildTransactionPayload.ts` — `TransactionFormValues` → `CreateTransactionPayload`
  - `apiErrors.ts` — `parseApiError`, `mapTagsToFieldErrors`
  - `formatCents.ts` — `formatCents`, `formatBalance`, `formatSignedCents`
  - `parseDate.ts` — `parseDate`, `localDateStr`, `convertUtcToLocalKeepingValues`
  - `groupTransactions.ts` — `groupTransactions(txs, groupBy, accounts, categories)`

## Naming Conventions

**Files:**
- React components: PascalCase (`TransactionForm.tsx`, `RecurrenceFields.tsx`)
- Hooks: camelCase prefixed with `use` (`useTransactions.ts`, `useCreateTransaction.ts`)
- API functions: camelCase (`transactions.ts`, `auth.ts`)
- Utilities: camelCase (`buildTransactionPayload.ts`, `queryKeys.ts`)
- Schemas: camelCase with `Schema` suffix (`transactionFormSchema.ts`, `importFormSchema.ts`)
- CSS Modules: same name as component (`TransactionRow.module.css`)

**Directories:**
- Lowercase, singular for resource domains (`transactions/`, `accounts/`, `categories/`)
- Lowercase for functional groupings (`form/`, `filters/`, `import/`)

## Where to Add New Code

**New API endpoint:**
- Add function to relevant `src/api/<domain>.ts`
- If new domain: create `src/api/<domain>.ts`

**New query (read):**
- Add hook to `src/hooks/use<Resource>.ts` returning `{ query, invalidate }`
- Add key to `src/utils/queryKeys.ts` first

**New mutation (write):**
- Add hook to `src/hooks/use<Action><Resource>.ts` returning `{ mutation }`
- Do not invalidate inside mutation hook; let the call site do it

**New route/page:**
- Add file `src/routes/_authenticated.<name>.tsx` (authenticated) or `src/routes/<name>.tsx` (public)
- TanStack Router plugin auto-updates `src/routeTree.gen.ts` on save
- Use `createFileRoute('/_authenticated/<name>')` with `validateSearch: zodValidator(schema)` for typed search params

**New domain types:**
- Add to `src/types/transactions.ts` inside the `Transactions` namespace if transaction-related
- For new domains: create `src/types/<domain>.ts` with a matching namespace

**New shared form field component:**
- Place in `src/components/transactions/form/` if transaction-specific
- Accept `namePrefix?: string` prop if the field will be reused in the import context
- Use `useFormContext<AnyFormValues>()` (cast to `any`) when the component must work in multiple form schemas

**New utility:**
- Add to `src/utils/<descriptiveName>.ts` as a named export pure function

**New drawer:**
- Create component in `src/components/<domain>/<Name>Drawer.tsx`
- Use `useDrawerContext<T>()` inside the component
- Open via `void renderDrawer(() => <MyDrawer />)` or `await renderDrawer<T>(() => <MyDrawer />)`
- Never use `useState`/`useDisclosure` to manage the drawer's own `opened` state

## Special Directories

**`src/routeTree.gen.ts`:**
- Purpose: Auto-generated TanStack Router route manifest
- Generated: Yes, by `@tanstack/router-plugin/vite` on save during dev
- Committed: Yes

**`e2e/`:**
- Purpose: Playwright tests; separate from `src/`
- `e2e/.auth/` — saved auth state for test sessions

**`.tanstack/tmp/`:**
- Purpose: TanStack Router plugin working directory
- Generated: Yes
- Committed: No (add to `.gitignore` if not already)
