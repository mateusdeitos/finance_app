**Test 1: `TestLinkedTransactionPropagation_DateDoesNotCrossToPartner`**

Setup:
1. Create two users (userA, userB) with accounts and accepted connection
2. Create category for userA
3. Create a recurring expense (3 monthly installments) with split settings for userA:
   ```go
   createReq := &domain.TransactionCreateRequest{
       Date:            time.Date(2026, 1, 15, 0, 0, 0, 0, time.UTC),
       Description:     "recurring shared expense",
       Amount:          10000,
       AccountID:       accountA.ID,
       TransactionType: domain.TransactionTypeExpense,
       CategoryID:      &categoryA.ID,
       RecurrenceSettings: &domain.RecurrenceSettings{
           Type:        domain.RecurrenceTypeMonthly,
           Repetitions: lo.ToPtr(3),
       },
       SplitSettings: []domain.SplitSettings{
           {ConnectionID: connection.ID, Percentage: lo.ToPtr(50)},
       },
   }
   created, err := suite.Services.Transaction.Create(ctx, userA.ID, createReq)
   ```
4. Get the linked (userB's) transaction from `created.LinkedTransactions[0]`
5. Record the partner's original date for later comparison

Update the linked transaction's date with propagation=all:
```go
newDate := time.Date(2026, 1, 20, 0, 0, 0, 0, time.UTC) // shifted 5 days forward
updateReq := &domain.TransactionUpdateRequest{
    Date:                &newDate,
    PropagationSettings: domain.TransactionPropagationSettingsAll,
}
err = suite.Services.Transaction.Update(ctx, created.LinkedTransactions[0].ID, userB.ID, updateReq)
suite.Require().NoError(err)
```

Verify:
- Fetch userB's transactions — all of userB's installments should have dates shifted by 5 days
- Fetch userA's transactions — userA's installment dates should be UNCHANGED (the partner's edit did not cross over)

```go
// userA's transactions should be unchanged
userATransactions, err := suite.Services.Transaction.GetByID(ctx, userA.ID, created.ID)
suite.Require().NoError(err)
suite.Equal(time.Date(2026, 1, 15, 0, 0, 0, 0, time.UTC).Day(), userATransactions.Date.Day(),
    "userA's transaction date should NOT have shifted")
```
