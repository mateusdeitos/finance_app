        id                SERIAL PRIMARY KEY,
        budget_id         INT     NOT NULL REFERENCES budgets(id) ON DELETE CASCADE,
        threshold_pct     INT     NOT NULL CHECK (threshold_pct BETWEEN 1 AND 200),
        enabled           BOOLEAN NOT NULL DEFAULT TRUE,
        last_fired_period TEXT,
        CONSTRAINT uq_budget_alert_thresholds_budget_pct UNIQUE (budget_id, threshold_pct)
    );
    CREATE INDEX idx_budget_alert_thresholds_budget_id ON budget_alert_thresholds(budget_id);

    - `UNIQUE (budget_id, threshold_pct)` — no duplicate threshold per cap.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| migration runner → PostgreSQL | DDL executed by Goose at deploy/migrate time. No request-time user input crosses here; SQL is static, authored DDL — not dynamic/parameterized at runtime. |
| (future) Phase 27 service → budgets table | Owner scoping (owner_user_id) will be enforced in Phase 27; flagged here, out of scope for this phase. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-01 | Tampering / Integrity | budgets table | mitigate | `UNIQUE(owner_user_id, category_id)` prevents duplicate caps per category (one-cap-per-category invariant, D-26-3). `NOT NULL` on owner_user_id, category_id, amount_cents, active guarantees no orphan/ambiguous cap rows. |
| T-26-02 | Tampering / Integrity | budget_alert_thresholds FK | mitigate | `budget_id REFERENCES budgets(id) ON DELETE CASCADE` guarantees no orphaned threshold rows after a cap is hard-deleted (D-26-4); `UNIQUE(budget_id, threshold_pct)` prevents duplicate thresholds. `CHECK (threshold_pct BETWEEN 1 AND 200)` bounds the value range. |
| T-26-03 | Information Disclosure / IDOR | budgets ownership | transfer (to Phase 27) | The `owner_user_id` column is the scoping key for IDOR enforcement; query-level owner scoping and 403-on-foreign-id are implemented in Phase 27's repository/service. Documented here so Phase 27 inherits the requirement. |
| T-26-04 | Denial of Service / migration safety | migrate-down rollback | mitigate | Symmetric Down blocks drop child (budget_alert_thresholds) before parent (budgets); each file drops only its own table, avoiding FK-violation failures on rollback (validated in Task 3, ROADMAP SC2). |
| T-26-05 | SQL Injection | DDL | accept | No dynamic SQL in this phase — all SQL is static authored DDL via Goose. No injection surface exists. |
</threat_model>
