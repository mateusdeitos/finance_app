## What Was Built

### ChargeSearchOptions and CreateChargeRequest (domain)

Two new domain types appended to `domain/charge.go`:
- `ChargeSearchOptions`: filters by UserID, Direction (sent/received/all), Status, ConnectionID with Limit/Offset and `query:` tags for Echo bind
- `CreateChargeRequest`: carries ConnectionID, Role (charger/payer), MyAccountID, PeriodMonth/Year, Description

### ChargeRepository

GORM implementation following `settlement_repository.go` pattern:
- `Create`, `GetByID`, `Update` — standard CRUD
- `Search` — direction-aware WHERE clause with `(charger_user_id = ? OR payer_user_id = ?)` for "all", single-column for "sent"/"received"; orders by `created_at DESC`
- `Count` — same filter logic, returns `int64` (for PendingCount badge)
- IDOR gate: `UserID > 0` required in caller; if zero, no user filter applied (repository trusts caller)

### ChargeService

Business logic with authorization checks:
- `Create`: validates role ("charger"|"payer"), fetches + validates connection (accepted, caller is party), resolves otherPartyID, sets ChargerUserID/PayerUserID based on role
- `Cancel`: IDOR check → role check (only charger) → transition validation → update
- `Reject`: IDOR check → role check (only payer) → transition validation → update
- `List`: delegates to repository Search, wraps errors
- `PendingCount`: calls Count with Direction="received", Status="pending" for badge counters

### Mocks

Both `mock_ChargeRepository.go` and `mock_ChargeService.go` generated by mockery v2.53.6.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 3 - Blocking] Import cycle broken: service types moved to domain**

- **Found during:** Task 3 (mock generation)
- **Issue:** Mockery v2.53.6 regenerated `mock_TransactionService.go` with `import service "github.com/finance_app/backend/internal/service"` because `ParseImportCSV` used `ImportDecimalSeparatorValue` and `ImportTypeDefinitionRule` types defined in the service package. This creates a cycle: `service/test_setup.go` → `mocks` → `service`.
- **Fix:** Moved `ImportDecimalSeparatorValue` and `ImportTypeDefinitionRule` types + constants from `service/structs.go` to `domain/transaction_import.go`. Updated `service/interfaces.go` and `service/transaction_import.go` to reference `domain.*` directly. Kept type aliases in `service/structs.go` so `transaction_handler.go` continues to compile unchanged (`service.DecimalSeparatorComma` still works via alias).
- **Files modified:** `domain/transaction_import.go`, `service/interfaces.go`, `service/structs.go`, `service/transaction_import.go`, `mocks/mock_TransactionService.go`
