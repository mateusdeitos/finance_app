## Spacing Scale

Declared values (Mantine spacing tokens, all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge gaps |
| sm | 8px | Avatar-to-text gap, compact element spacing |
| md | 16px | Card padding, section inner spacing |
| lg | 24px | Section padding, drawer sections |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Transfer arrow gap between avatars in transaction row: 4px (xs) — tight pairing to read as a unit
- Avatar touch target minimum: 44px height row (existing TransactionRow height applies; avatar itself can be smaller)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | Mantine default background (#ffffff / dark-mode surface) | Page background, card surfaces |
| Secondary (30%) | grey[0] (#f5f5f5) | Card borders, dividers, avatar background fallback |
| Accent (10%) | blue[7] (#457b9d) | Avatar ring color (current user), CTA buttons, active nav |
| Destructive | red[5] (#e63946) | Destructive actions only |

Accent reserved for: current-user Avatar `color` prop (in header, split settings), primary action buttons. Never used for informational states.

Avatar color mapping:
- Current user avatar: `color="blue"` (blue[7] = #457b9d) — source: existing AppLayout line 75
- Partner/connected user avatar: `color="grape"` — visually distinct from current user
- Private account initials avatar: user-chosen hex from ColorSwatch preset palette (stored in DB)
- Shared account avatar: shows partner OAuth photo; fallback `color="grape"`

ColorSwatch preset palette (12 colors, stored as hex in `accounts.avatar_background_color`):
`#e63946` (red), `#ef7819` (orange/yellow), `#2a9d8f` (teal), `#457b9d` (blue), `#6a4c93` (purple), `#e91e8c` (pink), `#2d6a4f` (forest), `#f4a261` (sand), `#457b9d` (steel), `#1a1a1a` (charcoal), `#8b8b8b` (grey), `#39b2a3` (mint)

Source: theme.ts color palette; colors chosen from existing theme tuples plus common distinct hues.

## Interaction Contract

### Transfer row — avatar pair

- Layout: `[source Avatar xs] [IconArrowRight size=12 opacity=0.5] [dest Avatar xs]`
- Each avatar wrapped in `<Tooltip label={accountName} withArrow>` — shows account name on hover (desktop) and tap (mobile)
- Tooltip `position="top"`, no delay on mobile (Mantine default touch behavior)
- Tooltip `multiline={false}` — account names are short single strings

### Non-transfer row — single avatar

- Layout: `[Avatar xs]` replacing the current text `<Text size="sm" c="dimmed">`
- Wrapped in `<Tooltip label={accountName} withArrow>` — same pattern as transfer

### Account card — avatar left

- Layout: `<Group gap="sm"><AccountAvatar ... size="md" /><Stack>(name + badges)</Stack></Group>`
- Private account: initials + user-chosen background color
- Shared account: partner OAuth photo; fallback to partner initials with `color="grape"`

### Header avatar — OAuth photo

- Layout unchanged except Avatar gains `src={user.avatar_url}`
- Mantine Avatar shows children (initials) automatically when `src` fails to load (onerror fallback built in)

### ColorSwatch picker in account form

- Rendered inside account create/edit drawer
- Grid layout: 4 columns × 3 rows, `gap={8}`
- Each swatch: 28×28px, `radius="xl"`, clickable
- Selected state: thin 2px ring using `box-shadow: 0 0 0 2px var(--mantine-color-blue-7)`
- Label above grid: "Cor do avatar" (pt-BR)
- Default selection: auto-assigned from migration (no null state per D-07)

## Copywriting Contract

All copy in pt-BR (Brazilian Portuguese), consistent with existing codebase.

| Element | Copy |
|---------|------|
| Tooltip — account name on avatar (transaction row) | `{account.name}` — dynamic, no wrapper label |
| Tooltip — account name on partner avatar (split settings) | `{selectedAccount.description ?? selectedAccount.name}` — existing pattern, unchanged |
| Color picker section label | "Cor do avatar" |
| Color picker aria-label per swatch | "Selecionar cor {hexCode}" |
| Empty avatar fallback | Initials (never empty — always name[0..1]) |
| Account card shared badge | "Compartilhada" — existing, unchanged |

No new empty states or error states introduced by this phase. Avatar load failures are silently handled by Mantine Avatar's built-in fallback to children (initials).

No destructive actions in this phase.
