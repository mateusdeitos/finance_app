## Context

The repo has a Go backend under `backend/` but no frontend. We need to create `frontend/` from scratch using the agreed-upon stack: Vite, React 19, TypeScript, TanStack Router, TanStack Query, React Hook Form, and Mantine.

## Goals / Non-Goals

**Goals:**
- Scaffold a working `frontend/` project that builds and runs
- Configure all stack dependencies correctly
- Wire up providers (TanStack Query, Mantine) in `main.tsx`
- Demonstrate end-to-end routing with a single Hello World page at `/`

**Non-Goals:**
- Any real feature pages (auth, transactions, etc.)
- API integration
- Testing setup
- CI/CD configuration

## Decisions

### File-based routing with TanStack Router
TanStack Router supports file-based routing via `@tanstack/router-vite-plugin`, which auto-generates the route tree from files under `src/routes/`. This avoids boilerplate and aligns with how the router plugin is typically used. Alternative: manual `createRouter` — rejected because file-based is cleaner and scales better.

### Route protection via `beforeLoad`
Protected routes use TanStack Router's `beforeLoad` hook to call the current-user endpoint and redirect unauthenticated users. This is preferred over a `PrivateRoute` wrapper component as it integrates with the router lifecycle and avoids conditional renders.

### CSS Modules for custom styling
Mantine provides the base design system. Any component-specific overrides are done in colocated `.module.css` files. No global CSS framework (Tailwind, etc.) is added to keep the styling approach consistent.

### `frontend/` at repo root
Placing the frontend at `frontend/` mirrors the existing `backend/` directory, keeping a clean monorepo structure.

## Risks / Trade-offs

- **React 19 + Mantine compatibility** → Mantine v7+ supports React 19; pin to v7.
- **TanStack Router codegen on first run** → The route tree file (`src/routeTree.gen.ts`) is auto-generated by the Vite plugin; it must be committed or gitignored consistently. We commit it to avoid requiring a build step just to get types.
