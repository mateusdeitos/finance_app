# Frontend Migration Plan тАФ Aligning with Updated `frontend/CLAUDE.md`

## Context

`frontend/CLAUDE.md` was recently updated with a set of mandatory conventions (thin routes, pages in `src/pages/`, caller-owned invalidation, `select` generic on query hooks, zero `useEffect` in components, drawers via `renderDrawer`, testid-only E2E selectors, no `any`). Several areas of the existing codebase predate these conventions and diverge. The "Known divergences" list at the bottom of the file catalogs them but says _agents should not copy them and should migrate when touching_. We need an explicit, ordered migration plan so the divergence list can shrink predictably instead of drifting.

Goal: ship the migration as **6 independent PRs plus an incremental E2E testid sweep**, each small enough to review, ordered so later phases consume infrastructure introduced by earlier ones. No big-bang rewrites.

## Non-goals (do NOT touch)

- `src/routeTree.gen.ts` тАФ regenerated by TanStack Router; never hand-edit (the `as any` there is generated).
- `src/utils/renderDrawer.tsx` тАФ already correct per spec ┬з8; treat as frozen API.
- `src/api/*` тАФ already pure fetch with no React imports.
- `src/utils/queryKeys.ts` тАФ nothing in this plan adds new keys.
- `src/types/*` namespace layout тАФ extend, don't restructure.
- `src/routes/__root.tsx`, `src/routes/_authenticated.tsx`, `src/routes/index.tsx` тАФ already thin.
- `src/hooks/usePWAInstall.ts` тАФ `useEffect` is already encapsulated in a hook (spec explicitly allows this).
- `useDisclosure` inside popovers (`ImportCSVBulkToolbar.tsx:77`) or collapsible UI state (`CategoryCard.tsx:42-43`) тАФ `renderDrawer` is for drawers/modals only.

## Reference implementations (mirror these shapes)

- `src/hooks/useAccounts.ts` тАФ canonical `{ query, invalidate }` + `select?: (data) => T` generic. Copy for `useTransactions`, `useCharges`.
- `src/hooks/useCreateAccount.ts` тАФ canonical mutation hook: returns `{ mutation }`, accepts `options?: { onSuccess?: () => void }`, no hardcoded invalidation.
- `src/components/transactions/SelectCategoryDrawer.tsx` тАФ canonical `renderDrawer` consumer with `useDrawerContext<T>()` + `close(value)`.
- `src/components/transactions/form/transactionFormSchema.ts` тАФ `baseTransactionFields` + `applySharedRefinements` pattern for shared Zod fields.

## Phase 1 тАФ Foundational hook alignment (small PR, blocks Phases 3тАУ5)

**Why first:** every later phase that touches a page/drawer will want to use `select` on `useTransactions`/`useCharges` and will want to own its own invalidation. Doing this first avoids rewriting the same callsites twice.

**Tasks:**

- `src/hooks/useTransactions.ts` тАФ add `<T = Transactions.Transaction[]>` generic + `select?: (data: Transactions.Transaction[]) => T`, forward to `useQuery`. Mirror `useAccounts.ts`.
- `src/hooks/useCharges.ts` тАФ same transformation.
- `src/hooks/useChargesPendingCount.ts`, `src/hooks/useInviteInfo.ts`, `src/hooks/useOpeningBalance.ts` тАФ add the `select` generic. Low churn, gets them uniformly compliant.
- `src/hooks/useCreateTransaction.ts` тАФ remove the hardcoded `queryClient.invalidateQueries` for `Transactions` + `Tags` in `onSuccess` (violates spec ┬з3: "invalidation is the caller's responsibility"). Keep `options.onSuccess?.()`. Drop `useQueryClient` import if unused.
- `src/hooks/useUpdateTransaction.ts` тАФ add `options?: { onSuccess?: () => void; onFieldErrors?: (errors: Record<string, string>) => void }` and wire `onSuccess`. Match the post-cleanup shape of `useCreateTransaction`.
- **Caller audit** тАФ grep `useCreateTransaction(` and `useUpdateTransaction(` and add `onSuccess: invalidate` (pulled from the local `useTransactions()`/`useTags()` call) at every callsite. Known callers: `CreateTransactionDrawer.tsx`, `UpdateTransactionDrawer.tsx`, `ImportReviewRow.tsx`, `_authenticated.transactions_.import.tsx`.

**Verify:** `npm run lint`, `npm run build`; E2E `transactions.spec.ts`, `update-transaction.spec.ts`, `bulk-update-transfer.spec.ts`, `recurrence.spec.ts`, `import.spec.ts` тАФ regressions surface here if a caller is missed.

  - In `CategoriesPage`, replace `useState<Category | null>` + `useDisclosure` with a single `await renderDrawer(() => <DeleteCategoryModal category={cat} allCategories={categories} />)` handler, branching on `result.action === 'confirmed'` to call `deleteMutation.mutate(...)`.

## Cross-cutting recommendations

- **ESLint enforcement** тАФ after Phase 4, add a `no-restricted-syntax` rule banning `useEffect` in `src/components/**` and `src/pages/**`. After Phase 6, flip `@typescript-eslint/no-explicit-any` to `error`. These make the conventions self-enforcing and stop regressions.
- **Keep the "Known divergences" section in `frontend/CLAUDE.md` accurate** тАФ each merged phase PR removes the corresponding bullet. When the list is empty, delete the section.

- **Convention audits** (as part of each PR review):
  - `grep -rn "useEffect" src/components src/pages src/routes` returns zero after Phase 4.

  - `grep -rn ": any\|as any\|<any>" src/ | grep -v routeTree.gen.ts` returns zero after Phase 6.

- **Bundle/runtime smoke:** `npm run dev`, click through Accounts тЖТ Categories тЖТ Transactions тЖТ Import тЖТ Charges тЖТ Connect-with; open and dismiss every drawer by swipe, ESC, and backdrop.
